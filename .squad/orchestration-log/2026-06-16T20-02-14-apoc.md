## Summary

Root cause: in-process ONNX GenAI/WinML CUDA KV-cache arena sizes to each request's INPUT prompt length and is never released for process lifetime (high-water-mark). UI resent the full growing transcript with no max_tokens, driving arena to OOM.

## Changes

1. **New FoundryLocalOptions:**
   - `MaxPromptTokens` (default 1024)
   - `MaxResponseTokens` (default 2048)

2. **New OpenAiChatHelpers.ApplyContextBounds:**
   - Caps max_tokens
   - Trims oldest turns, preserving system + latest user
   - Head-truncates oversized latest message
   - Wired into /v1/chat/completions

3. **Hardened EnsureModelLoadedAsync:**
   - Made idempotent with IsLoadedAsync guard
   - Unload-others-before-reload prevents duplicate copies

4. **New RepeatedPromptVramTests:**
   - Playwright-driven, samples nvidia-smi
   - Verifies peak ≤ 5000 MiB, growth < 2500 MiB
